What is the Edit Field (Set) Node in n8n?

The Edit Field (Set) node is one of the most fundamental and frequently used nodes in n8n. Its primary purpose is to manipulate data flowing through your workflow.

Think of it as a data-shaping tool: it allows you to create new fields, update existing ones, remove clutter, or completely rewrite the data structure passing from one node to the next.

Key Use Cases

  • Adding New Fields: Create a brand new variable or key-value pair (e.g., combining firstName and lastName into fullName).
  • Modifying Existing Data: Update the value of a field that already exists (e.g., converting text to lowercase or formatting a date).
  • Data Type Conversion: Explicitly cast data types, such as turning a string "125" into a proper number 125.
  • Cleaning Up Payload (Removing Fields): Keep only the specific fields you need and drop the rest to keep your workflow clean and fast.

The Core Modes

  • Manual Mapping: Explicitly define individual fields, their types, and assign static or dynamic values.
  • Remove Fields: Specify a list of field names that you want to delete from the incoming JSON payload.
  • Keep Only Set Fields (Toggle): When turned on, the node wipes out the entire incoming payload and only outputs the exact fields you explicitly defined inside the node.
